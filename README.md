
Ниже собрал **user-flow сервиса**, плюс **варианты использования** и **сценарии** так, чтобы это можно было почти сразу перекладывать в продуктовый документ, CJM или структуру прототипа.

  

# **User-flow сервиса**

  

## **1. Точка входа**

  

Пользователь попадает в сервис из рекламы, соцсетей, рекомендации, статьи, карты, партнерской ссылки или прямого захода.

  

### **Что он видит первым экраном**

  

Не каталог мест, а понятное обещание результата:

  

**“Подберём маршрут по Краснодарскому краю под ваши интересы, даты, бюджет и формат отдыха”**

  

Кнопки входа:

- **Подобрать поездку**
    
- **Посмотреть готовые идеи**
    
- **Для бизнеса**
    

  

Это важно, потому что у пользователя обычно один из двух режимов:

- он уже хочет что-то подобрать под себя
    
- он пока просто вдохновляется и не готов заполнять длинную анкету
    

---

## **2. Ветка выбора сценария входа**

  

### **Вариант A. “Подобрать поездку”**

  

Основной персонализированный flow.

  

### **Вариант B. “Посмотреть готовые идеи”**

  

Более лёгкий вход для холодного трафика:

- винные выходные
    
- поездка с детьми
    
- фермы и гастро
    
- однодневные поездки
    
- без машины
    
- для пары
    
- спокойный отдых 60+
    

  

Из этой ветки пользователь потом всё равно может перейти в персонализацию.

  

### **Вариант C. “Для бизнеса”**

  

Отдельный b2b-flow:

- зачем партнёру подключаться
    
- как место попадает в маршруты
    
- как работает комиссия / лиды / акции
    
- регистрация партнёра
    

---

# **Основной пользовательский flow**

  

## **Шаг 1. Квиз интересов**

  

Пользователь проходит короткий сценарий выбора.

  

### **Экран 1. Цель поездки**

  

Примеры опций:

- отдохнуть и переключиться
    
- гастро и фермы
    
- вино и дегустации
    
- природа и прогулки
    
- семейная поездка
    
- романтический выходной
    
- поездка с родителями
    
- активный день
    
- уединение
    
- workation
    

  

### **Экран 2. С кем едет**

- один
    
- пара
    
- семья с маленькими детьми
    
- семья с подростками
    
- компания друзей
    
- с родителями / старшим поколением
    
- смешанная группа
    

  

### **Экран 3. Даты**

- конкретные даты
    
- ближайшие выходные
    
- без точной даты
    
- 1 день / 2 дня / 3+ дней
    

  

### **Экран 4. Транспорт**

- на машине
    
- без машины
    
- нужна помощь с трансфером
    
- готов арендовать авто
    

  

### **Экран 5. Бюджет**

- эконом
    
- средний
    
- комфорт
    
- премиум
    

  

Лучше не только ползунок, а понятные диапазоны на человека / на поездку.

  

### **Экран 6. Темп отдыха**

- насыщенно
    
- спокойно
    
- сбалансированно
    

  

### **Экран 7. Важные условия**

  

Теги:

- можно с детьми
    
- без долгих переездов
    
- без сложных маршрутов
    
- нужен интернет
    
- можно с питомцем
    
- нужна ночёвка
    
- желательно с едой на месте
    
- желательно с дегустацией
    
- красиво для фото
    
- без толп
    

  

### **Экран 8. Уровень персонализации**

  

Можно добавить лёгкий вопрос:

**“Что для вас важнее?”**

- новые впечатления
    
- комфорт и предсказуемость
    
- красивые виды
    
- еда и вкус
    
- тишина
    
- насыщенная программа
    

  

Это поможет ранжированию.

---

## **Шаг 2. Экран “Мы собираем сценарии”**

  

Тут как раз можно вставить микро-вау:

- не просто спиннер
    
- а короткую сборку будущего маршрута
    

  

Например:

**“Подбираем спокойные маршруты без длинных переездов”**

**“Учитываем винодельни, гастро-точки и доступность в ваши даты”**

**“Ищем варианты под поездку для пары”**

  

Это уже создаёт ощущение персональной работы.

---

## **Шаг 3. Выдача 2–3 сценариев поездки**

  

Это ключевой экран.

  

Не список из 30 мест, а **3 карточки-сценария**.

  

Например:

- **Тихий винно-гастро выходной**
    
- **Однодневная поездка к природе и ферме**
    
- **Спокойный маршрут у моря с ужином и ночёвкой**
    

  

### **Каждая карточка должна содержать**

- название сценария
    
- 1 короткое объяснение “почему вам подходит”
    
- длительность
    
- примерный бюджет
    
- темп
    
- формат логистики
    
- кому подходит
    
- 3–4 ключевые точки
    
- CTA: **Посмотреть сценарий**
    

  

### **Здесь же можно вставлять preview**

  

Не полноценный длинный ролик, а короткий интерактивный teaser:

- фоновые видео / фото
    
- подписи
    
- “утром — дорога”, “днём — дегустация”, “вечером — ужин”
    
- кнопка “развернуть день”
    

---

## **Шаг 4. Детальный просмотр сценария**

  

Пользователь открывает один сценарий и видит уже не просто концепт, а почти готовый продукт.

  

### **Блок 1. Эмоциональный preview**

  

Формат:

- видеоряд / слайд-стори
    
- интерактивные надписи
    
- описание атмосферы
    

  

Пример:

**“Этот маршрут подойдёт для пары, которая хочет провести день без спешки, с красивыми видами, дегустацией и ужином вечером”**

  

### **Блок 2. Логика дня**

  

По таймлайну:

- 09:00 — выезд
    
- 11:00 — первая точка
    
- 13:30 — обед / ферма
    
- 16:00 — прогулка / дегустация
    
- 19:00 — ужин / заселение
    

  

### **Блок 3. Карта и переезды**

- точки маршрута
    
- длительность между ними
    
- общий километраж
    
- удобно / не очень удобно без машины
    

  

### **Блок 4. Почему этот вариант подходит**

- без длинных переездов
    
- спокойный темп
    
- подходит новичкам в винном туризме
    
- можно с детьми / не подходит детям
    
- хорошие активности в межсезонье
    

  

### **Блок 5. Стоимость**

- минимальный бюджет
    
- комфортный бюджет
    
- что входит / не входит
    

  

### **Блок 6. Риски и ограничения**

  

Это очень важно для доверия.

- нужна предварительная запись
    
- в дождь часть маршрута теряет смысл
    
- в высокий сезон больше людей
    
- без машины сложнее
    
- детям младше X может быть скучно
    

  

### **Блок 7. Альтернативы**

- если плохая погода
    
- если нет машины
    
- если хочется дешевле
    
- если хочется больше природы
    
- если хочется ночёвку
    

  

### **Блок 8. CTA**

- сохранить маршрут
    
- получить ещё 2 похожих
    
- открыть доступность / бронирование
    
- отправить себе / другу
    
- собрать свой вариант на базе этого сценария
    

---

# **Шаг 5. Переход к бронированию или сборке маршрута**

  

После просмотра сценария пользователь должен иметь три пути.

  

## **Путь A. “Забрать как есть”**

  

Подходит тем, кто хочет быстрое решение:

- забронировать точки
    
- получить финальный план
    
- отправить в мессенджер
    
- открыть на карте
    

  

## **Путь B. “Подстроить под себя”**

  

Тонкая настройка:

- убрать одну точку
    
- добавить ночёвку
    
- заменить винодельню на ферму
    
- сделать дешевле
    
- сократить поездку
    
- сделать маршрут для детей
    

  

## **Путь C. “Посмотреть похожие”**

  

Для тех, кто пока сравнивает:

- больше гастро
    
- больше природы
    
- без вина
    
- без машины
    
- дешевле
    
- ближе
    
- спокойнее
    

---

# **Пост-конверсионный flow**

  

После того как пользователь сохранил маршрут или что-то забронировал, начинается очень важный слой retention.

  

## **После сохранения маршрута**

- маршрут в избранном
    
- напоминание перед поездкой
    
- изменения по погоде / событиям
    
- рекомендации “что добавить рядом”
    
- сценарии на другой сезон
    

  

## **После поездки**

- просьба оценить не место отдельно, а весь сценарий
    
- вопрос “что оказалось лучше / хуже ожиданий”
    
- предложение похожих маршрутов
    
- персональные подборки на будущее
    

  

Это важно, потому что у тебя продукт не про единичную транзакцию, а про повторные поездки по сезону.

# **Варианты использования**

  

Теперь разложу не по экранам, а по типам поведения.

  

## **1. Пользователь не знает, куда хочет ехать**

  

Самый частый кейс.

  

### **Его задача**

  

Понять, какие вообще есть варианты под него.

  

### **Flow**

- заходит
    
- проходит квиз
    
- получает 2–3 сценария
    
- смотрит teaser
    
- открывает 1–2 сценария
    
- сохраняет или уходит на обсуждение с семьёй / парой
    

  

### **Что важно**

  

Ему нужен не каталог, а быстрое сужение выбора.

---

## **2. Пользователь уже хочет формат, но не знает точку**

  

Пример:

“хочу винный выходной”

или

“хочу куда-то с детьми на один день”.

  

### **Flow**

- выбирает категорию сразу на первом экране
    
- уточняет состав группы и бюджет
    
- получает сценарии внутри нужного формата
    
- сравнивает
    

  

### **Что важно**

  

Дать ему ощущение, что сервис понимает не только тему, но и контекст.

---

## **3. Пользователь выбирает между несколькими сценариями**

  

Он уже почти готов, но сомневается.

  

### **Flow**

- открывает 2–3 сценария
    
- смотрит preview
    
- сравнивает:
    
    - бюджет
        
    - логистику
        
    - темп
        
    - сезонность
        
    
- выбирает один
    

  

### **Что важно**

  

Нужен режим сравнения сценариев. Это сильная функция.

---

## **4. Пользователь хочет быстрое решение “на выходные”**

  

Очень ценный сценарий.

  

### **Flow**

- нажимает “на ближайшие выходные”
    
- указывает город выезда
    
- отмечает “не хочу долго планировать”
    
- получает 2 готовых маршрута
    
- выбирает и сохраняет
    

  

### **Что важно**

  

Максимум скорости, минимум настроек.

---

## **5. Пользователь хочет вдохновиться без готовности к покупке**

  

Он ещё холодный.

  

### **Flow**

- смотрит готовые идеи
    
- открывает красивые сценарии
    
- вовлекается через preview
    
- потом идёт в квиз ради персонализации
    

  

### **Что важно**

  

Это контентный вход. Нужен для трафика из соцсетей и рекламы.

---

## **6. Пользователь возвращается повторно**

  

Очень сильный сегмент.

  

### **Flow**

- сервис узнаёт его прошлые интересы
    
- предлагает новые сезонные маршруты
    
- учитывает прошлые сохранения
    
- показывает:
    
    - “вам понравились винодельни — вот весенние гастро-маршруты”
        
    - “в прошлый раз вы выбирали спокойные поездки без долгих переездов”
        
    

  

### **Что важно**

  

Повторный визит должен быть умнее первого.

# **Сценарии использования**

  

Теперь в формате более живых продуктовых сценариев.

  

## **Сценарий 1. Пара на выходные**

  

### **Контекст**

  

Пара из Краснодара хочет куда-то выбраться на субботу-воскресенье.

  

### **Шаги**

- выбирают “романтический выходной”
    
- состав: пара
    
- транспорт: машина
    
- бюджет: средний
    
- темп: спокойно
    
- получают 3 сценария
    
- смотрят teaser по Абрау-Дюрсо
    
- видят: прогулка, дегустация, ужин, ночёвка
    
- сохраняют маршрут
    
- переходят к бронированию
    

  

### **Что их цепляет**

  

Не “список мест”, а готовая картинка выходных.

---

## **Сценарий 2. Семья с детьми**

  

### **Контекст**

  

Нужен безопасный и не утомительный маршрут на день.

  

### **Шаги**

- выбирают “семейная поездка”
    
- семья с маленькими детьми
    
- без долгих переездов
    
- нужен обед и туалет / парковка
    
- получают маршруты:
    
    - ферма + эко-локация
        
    - природа + мастер-класс
        
    
- открывают сценарий
    
- видят, что везде понятная инфраструктура
    
- принимают решение
    

  

### **Что их цепляет**

  

Снижение тревоги и понятность.

---

## **Сценарий 3. Компания друзей**

  

### **Контекст**

  

Хочется весело, вкусно, красиво, но без долгого планирования.

  

### **Шаги**

- выбирают “друзья”
    
- интересы: вино, еда, виды
    
- бюджет: комфорт
    
- темп: насыщенно
    
- получают 3 маршрута
    
- смотрят preview
    
- сравнивают
    
- отправляют один маршрут в общий чат
    

  

### **Что их цепляет**

  

Возможность быстро показать друзьям готовую идею.

---

## **Сценарий 4. Поездка для родителей 60+**

  

### **Контекст**

  

Нужен спокойный и предсказуемый маршрут.

  

### **Шаги**

- выбирают “с родителями”
    
- отмечают “без сложных дорог”, “спокойный темп”
    
- получают варианты с короткими переходами
    
- видят блок “почему этот маршрут подходит”
    
- выбирают комфортный вариант
    

  

### **Что их цепляет**

  

Не эмоция вау, а ощущение надёжности.

---

## **Сценарий 5. Пользователь без машины**

  

### **Контекст**

  

Хочет выбраться, но транспорт ограничивает.

  

### **Шаги**

- отмечает “без машины”
    
- система фильтрует всё неудобное
    
- показывает только реально достижимые сценарии
    
- даёт трансфер / такси / электричка, если возможно
    
- пользователь не сталкивается с ложным выбором
    

  

### **Что его цепляет**

  

Честность и реалистичность.

---

## **Сценарий 6. Workation / удалёнщик**

  

### **Контекст**

  

Хочет совместить отдых и работу.

  

### **Шаги**

- выбирает “работаю удалённо”
    
- важные условия: интернет, тишина, проживание, кофе / еда
    
- получает сценарии 2–4 дня
    
- видит не только красивые кадры, но и практичность
    
- бронирует
    

  

### **Что его цепляет**

  

Комбинация атмосферы и утилитарности.

---

# **Альтернативные продуктовые ветки**

  

## **Ветка “Собрать самому”**

  

Для части пользователей нужен полу-конструктор.

  

Flow:

- выбрать базовый сценарий
    
- заменить одну точку
    
- добавить ещё одну остановку
    
- изменить темп
    
- перестроить логистику
    

  

Это полезно, но я бы не делал это главным входом в MVP.

  

## **Ветка “AI-консьерж”**

  

После выдачи сценариев:

- “сделай маршрут дешевле”
    
- “убери винодельни”
    
- “добавь место для ужина”
    
- “сделай маршрут без ночёвки”
    

  

Это сильная премиальная фича.

  

## **Ветка “Посмотреть поездку как историю”**

  

Формат сторис:

- экран 1: для кого маршрут
    
- экран 2: что почувствуете
    
- экран 3: что будет по шагам
    
- экран 4: сколько стоит
    
- экран 5: забрать маршрут
    

  

Это очень хорошо для mobile-first.

---

# **Где именно вставлять видеоряд**

  

Я бы вставлял его в трёх местах.

  

## **1. После квиза**

  

Короткий teaser, пока “идёт подбор”.

Цель — усилить предвкушение.

  

## **2. В карточке сценария**

  

Мини-preview на 5–10 секунд.

Цель — поднять CTR в открытие сценария.

  

## **3. Внутри детального сценария**

  

Уже полноценный интерактивный preview.

Цель — дожать до сохранения / бронирования.

  

Главное: не ставить видео раньше, чем понятен персональный контекст. Иначе это будет просто красивая реклама.

# **Какие метрики покажут, что flow работает**

  

Я бы смотрел вот на это:

  

### **Верх воронки**

- старт квиза
    
- completion rate квиза
    
- drop-off по шагам
    

  

### **Центральная часть**

- open rate сценариев
    
- доля пользователей, открывших 2+ сценария
    
- время просмотра preview
    
- CTR из preview в детали
    

  

### **Низ воронки**

- сохранение маршрута
    
- share маршрута
    
- переход к бронированию
    
- подтверждённая бронь
    

  

### **Качество рекомендаций**

- процент сценариев, которые пользователь не отбрасывает сразу
    
- частота “показать похожее”
    
- частота ручной пересборки

# **Как бы я собрал MVP-flow**

  

Для MVP я бы не распылялся. Оставил бы такой путь:

  

## **MVP версия**

1. Первый экран с обещанием результата
    
2. Квиз 6–7 шагов
    
3. Выдача 3 сценариев
    
4. Внутри каждого:
    
    - 1 preview
        
    - таймлайн дня
        
    - логистика
        
    - бюджет
        
    - кому подходит
        
    - CTA “сохранить”
        
    
5. После сохранения — заявка / бронь / контакт с партнёром
    

  

Этого уже достаточно, чтобы проверить главное:

**цепляет ли персональный сценарий сильнее, чем обычный каталог мест.**

# **Сжатая схема всего user-flow**

  

**Вход**

→ подбор / готовые идеи / для бизнеса

  

**Подбор**

→ интересы

→ состав группы

→ даты

→ транспорт

→ бюджет

→ темп

→ важные условия

  

**Сборка**

→ экран ожидания с персональными сообщениями

  

**Выдача**

→ 2–3 сценария

→ teaser-preview

→ объяснение “почему подходит”

  

**Детали сценария**

→ эмоциональный preview

→ таймлайн

→ карта

→ бюджет

→ ограничения

→ альтернативы

  

**Действие**

→ сохранить

→ поделиться

→ настроить

→ забронировать

  

**После**

→ напоминания

→ обновления

→ похожие сценарии

→ повторные поездки
